Introduction

This HOWTO will go through the basics of creating a model (starfighter or otherwise) and then playing as it or against it in vegastrike.

Note: This HOWTO is a work in progress, chances are there are going to be a lot of missing or wrong bits. If anyone can spot an error, or would like something included, let someone know.

If anyone would like to change/improve this, feel free to do so.....so long as you send any modifications to me.

To get started you are going to need several things.

1. A copy of Vegastrike

2. A 3D modelling program that ouputs 3DS or OBJ files, or a converter that does the same.

3. Access to a 'perl' enabled computer

4. An xml converter script

Optional 5. A copy of Texturiser

Note: It is not necessary to get the cvs version of Vegastrike, the tarball should work fine. Texturiser however has had a few new features added since the last "pre-release", and some of these will be referred to in this howto. So if you plan to use texturiser with your modelling, it is recommended --but not nescesary-- that you get the cvs version.

Doing it -"I've got everything, where do I go from here"

  • Modelling (General)
If you are new to modelling, then you will need to learn to use your chosen program first. In my opinion, tutorials are the best way to do this. The selection in this howto is very limited and was not the result of a specific search. If no tutorials can be found in the program, or on your programs website, use a general search engine to look -there will almost certainly be some lurking around somewhere.
    Once you have a basic understanding of how your program works, you are ready to begin creating. There are several key things you must think about before creating a ship for vegastrike. First of all, you need to be able to export into either the 3DS or OBJ file types. If your program cannot do this, get a converter that can.
    Secondly, you are making a ship for a game, so there are going to be many ships being rendered at any one time --this means you are going to have to try to keep a low polygon count (1-400 maximum for a fighter craft, and nothing more than 1000 for the very largest). This is assuming you are prepared to make multiple LODs (Level Of Detail meshes) for your craft, if you are not going to, the polygon count should be much smaller. I will get into that later.
    Only use the size texture file that you need, there is no sense having a huge texture file for your ship when you only use a few parts of it --be economical! If your program or filetype converter doesn't export UV co-ordinates (specifying where polygons get their texture), then you are going to have to use texturiser for texuturing. This means you will have to texture every polygon seperately...... a lot of work for the bigger craft.

    IMPORTANT:  from versions 0.2.2 and later of Vegastrike. textures must have sizes that are powers of two (ie 64, 128, 256 etc) or else the game will crash upon loading the texture.
    That's all you really need to know at this stage, if it seems harder than you thought to make a decent looking ship, it's because it is. But with some imagination and determination, you should get the results you want.
  • Converting (Getting to a VS supported file type)
If your chosen program can output into of the supported file types, then there is no need for you to read this stage. If not though, then you need a converter that is able to convert your spaceship file into a supported file. Simply follow the converters instructions to export your model. If you do not want to have to retexture the whole craft again (assuming you did it already), then you should make sure that your converter can convert UV coordinates. If it doesn't and you can't find one that does (Anim8or should do it), don't despair. All it means is that you are going to have to retexture you craft (and if you have any, its LODs) using texturiser.
  • Converting (Getting to xml)

Note: If you are running linux, then you should have perl already installed (or at least on your distribution CDs). If you are using windows, you most likely do not have it -so follow the instructions here .

Linux: Once perl is installed it is all very easy. The converters are perl scripts, so simply apply the script to a file.....

Eg using bash: ./3ds2xml epeellcat.3ds

Windows: In windows it is a bit harder but still workable.

For windows: Read the mini-howto here.

If all went well, the script should have run without a hitch and you should now have one or more *.xmesh files (with the UV coordinates intact) and two *.xunit files. If not, then either you, or one of your programs, stuffed up. Occasionally, the output file from your program may not work (although you know it to be completely valid), in this case, try exporting again from another program.

You can safely delete all but one xunit file, and then rename it to what your ship is called (while lopping off the xunit extension). Now, open up the unit file, and make sure that it is pointing to the right xmesh file/s.

For a single mesh file: <Meshfile file="epeellcat.xmesh"/>

For multiple mesh files:

<Meshfile file="epeellcat1.xmesh"/>

<Meshfile file="epeellcat2.xmesh"/>

<Meshfile file="epeellcat3.xmesh"/>

And then open up the xmesh file, to make sure it's pointing to the right texture file.

Congratulations, your basic ship designing is done (unless you need to texture using texturiser). Now it is time to change your ship from an empty shell to a fully fledged ship, using the xmesh and unit files. Texturiser is good to be used when adding weapons, but is not nescesary. Other options are available for your model other than a simple texture. The options and how to use them are found in the xmesh section of this howto.

Texturiser -Preparing to texture

The texturiser is an utility for adding weapons and engines etc to a ship, as well as changing the uv mapping of a texture on a mesh. In the editor (because it's an old branch of vegastrike) confed stars and blue turret-like things are used to place weapons etc.

You texture the unit titled texture.xunit .You rename the .xunit file of the spaceship you wish to texture as "texture.xunit". If there is no texture on the .xmesh file you can specify it either by writing it in texture.txt, or by editing the first line of the .xmesh file so that the first line reads

<Mesh texture="xxxxx">

Where xxxxx is the name of the texture file ie epeellcat.png. The file types that are currently supported are bmp and png. Although it is prefered, if your ship is going to be submitted, that it be in the png filetype (and compressed as much as possible).

Ok now it is time to run the program. To get a feel for this, it is recommended that you follow the walkthrough, which will take you through the process of texturing and adding weapons . You can also refer to the controls page for a printable list of all texturiser commands.

The xmesh file -Pretty decorations and such...

The options in the xmesh file are pretty much straight forward. The top line as you would notice starts with <Mesh texture="yourtexture.png">. There are other options available though, these are scale, animation, blendmode and alphamap.

Using BMP files:

Scale:0.5 is half scale, 1.0 is normal etc

Animation:Allows texture animations. eg animation="test.ani" The is a plain text file using the following format:

numframes numsecondsperframe

frame1.bmp frame1.alp

frame2.bmp frame2.alp

The alp files are optional 256 colour greyscale files specifying the transparency. White is solid, and black is transparent.

Blendmode:This is the transparency of the mesh. If it is set to "ONE ZERO" then the mesh will be solid. If it is "ONE ONE" then it will be 50% transparent. If it is set to "SRCALPHA INVSRCALPHA" then it will use the alphamap file to set the transparency.

Alphamap:This is the map to be used to set the transparency with blendmode="SRCALPHA INVSRCALPHA". It should be in the same format as the alp files (256 colour greyscale) and is used in the same way, with white solid, and black transparent.

Using PNG files:

Sorry, not yet :-(

You can also add some other options at the bottom of the file:

</Polygons>

<material reflect="0">

<specular red="0" green="0" blue="0" alpha="0"/>

<ambient red="1" green="1" blue="1" alpha="1"/>

<diffuse red="1" green="1" blue="1" alpha="1"/>

</material>

</mesh>

These are all pretty self explanatory, but if you don't get it, feel free to ask.

The unit file -Give your ship some guts!

The unit file is the file that defines what your spaceship can do. It is here that you can adjust its mass, speed, engines and most important of all...... weapons. Self explanatory items (of my judging) will not be covered. It is probably best to look at some of the unit files of the vegastrike ships, and then fiddle with the values not covered here until you get a result you are happy with.

Mounts have sizes, specifying which types of weapons they could hold should that ordinance be available. The sizes available are:

"light," "medium," "heavy," "capship-light," "capship-heavy" and "special"

and for missiles:

"light-missile," "medium-missile," "heavy-missile," "light-capship-missile," "heavy-capship-missile," "special-missile"

The mount can be any of these types, or if nothing is specified, the weapon will be fixed.

Now that mount size is specified, you can choose the weapon type. The names of all current weapons available to be specified is located in weapon-info.xml. If you want to, you are welcome to create your own weapons for inclusion. An example is:

<Mount weapon="Laser" size="Light" x="1.600000" y="-1.200000" z="14.000000" ri="0.762493" rj="0.457496" rk="0.457496" qi="-0.432829" qj="0.886268" qk="-0.164887"/>

Turrets are similar, and can have a file specified from where to load the turret unit (and meshes).

eg <Subunit file="turret" x="-37.414394" y="13.548389" z="204.285370" ri="0.762493" rj="0.457496" rk="0.457496" qi="-0.432829" qj="0.886268" qk="-0.164887"/>

In both of these cases, ri,rj,rk and qi,qj,qk are the orientation of the weapon. They appear when texturizer is used to place the gun/turret and point out perpendicular from the polygon it was placed on. Delete them to make the weapon face forward, or leave rk="-1" to make it face backwards.

Lights can have both color and a texture specified.

eg <Light file="texturefile.png" x="0" y="0" z="-66" size="50" r="0.898039" g="0.803922" b="0.070588"/>

Here, r,g,b is the colour of the light (each ranging from 0 to 1).

Cockpits can have the file specified and the position of the cockpit.

eg <Cockpit file="bomber-cockpit.cpt" x="10" y="10" z="100"/>

Playing against or as your ship

Well, this is the easy part. Create a folder in the units subdir with its name being your ships name. Now move the mesh, unit file and texture into it (the texture could also be place in the textures directory). Make sure that the unit file's name is the same as the folder's name and then we are done. Now simply open up the mission file you intend playing. You are positioned in the first flightgroup. Change the type of craft to your new one and off you go. If you want to play against your ship, then simply change any other flightgroup to use your ship.

Tip: If you find your ships blowing up before you can do anything, they may be too big for the current spacing........resize them in the xmesh file or move them away from other craft!

    Modellers
