Installing Vegastrike     to Index

Hardware requirements:

  • 200Mhz CPU or better
  • 3D accelerated graphics card
  • 128 MB RAM
    to be able to see the terrain, you should have 256MB RAM
  • optional:
    • (3D) sound card
    • joystick

Installing Vegastrike on Linux:

To be able to run Vegastrike on Linux, you need to have hardware-accelerated 3D rendering (mesa-software rendering will be too slow) installed.

The binary contains the libraries Vegastrike needs to run, please contact us if it doesn't work on your machine.
The binary needs glibc-2.2 to work.

This description is valid for Vegastrike-0.2.0:

1. go to the download page and download from there the latest unixdata tar file and the latest linux binary

2. extract the binary to a directory of your choice
    example: tar zxf ../vegastrike-0.2.0-x86-linux.tar.gz

3. cd to the created subdirectory
    cd vegastrike-0.2.0

4. extract the content from the unixdata tarball into this directory:
    tar zxf ../vegastrike_unixdata-0.2.0.tgz

5. then cd to the data directory

6. first, you have to configure Vegastrike to match your system/cpu/gfx card etc
    type
    ./vssetup

    The vssetup dialog will pop up where you can select your preferences.

7. to start Vegastrike with the Mission Selector, type
    ./vsselect

    A dialog will pop up that shows you a list of all available missions.
    Select one and then click 'run mission'  and Vegastrike will start

 8. you can also start Vegastrike from shell without using the Mission Selector
    type
    ./vegastrike <mission-name>

    Examples:  ./vegastrike mission/earth_patrol1.mission
                      ./vegastrike mission/ships_confed1.mission

    Use as argument any of the missions in ./mission

  Please send us Email if you're having any problems.
