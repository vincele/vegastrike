Vegastrike Player Tutorial - Editing vegastrike.config    to Index

The file vegastrike.config is the main config file for Vegastrike.

The vegastrike.config contains key- and joystick-bindings, color settings and variables which control the way the game works.

The vegastrike.config file can be found at data/vegastrike.config in your Vegastrike distribution.

On Unix: if you copy the vegastrike.config to ~/.vegastrike/vegastrike.config , the config file from your home directory will be used.

To make simple changes to vegastrike.config, you can use the vssetup tool.

If you want to have your changes more refined, you have to edit vegastrike.config yourself.
 
 

Editing vegastrike.config with an editor:

The vegastrike.config file is a XML-file - it should be editable by users that have heard of HTML or XML before.

The vegastrike.config file has to start with a <vegaconfig> tag, then followed by those sections:

  • bindings section
  • color section
  • variables section

Bindings section:

The bindings section starts with the tag <bindings>. It includes
  • key/joystick bindings
  • axis definitions
  • hatswitch definitions/bindings
Short example for a bindings-section:

<bindings>
   <bind key="space" modifier="none" command="FireKey" />
   <bind key="y" modifier="none" command="MatchSpeedKey" />
   <bind key="j" modifier="none" command="JumpKey" />
   <bind key="backspace" modifier="none" command="StopKey"/>
   <bind key="cursor-down" modifier="none" command="DownKey"/>
   <bind key="function-1" modifier="none" command="Cockpit::Inside"/>

  <bind joystick="0" button="0" modifier="none" command="FireKey" />
  <bind joystick="0" button="1" modifier="none" command="ABKey" />

  <axis name="x" joystick="0" axis="0" inverse="false" />
  <axis name="y" joystick="0" axis="1" inverse="false" />
 </bindings>

In the first part, keys on the keyboard are bound to commands.
Keys are either the character you have to press or a long name like 'backspace' or 'function-1' for special keys.
There are no modifiers yet.

Commands have names like 'FireKey' or 'Cockpit::Inside'

In the next part, joystick buttons are bound to commands.
'joystick' is the number of the joystick, 'button' the number of the button.

The last part are the axis bindings.
In the example, you tell the game to use axis 0 on joystick 0 to bank your ship left/right.

There are NO default settings compiled into the game - each key/button/axis has to be bound in the vegastrike.config file.
 
 

Color section:

The color section starts with a <colors> tag.

You can define here colors and lights, when you're not satisfied with the look of Vegastrike.
 
 

Variables section:

The variables section starts with a <variables> tag.

The variables section can have many sections. Sections are started with the <section> tag.

Each section may contain a number of variable definitions and sub-sections.

Variable definitions start with the keyword <var>.
Subsections have the keyword <section>, too.

Small example:

<variables>
   <section name="data">
       <var name="universe_path" value="universe"/>
       <var name="sharedtextures" value="textures"/>
       <var name="sharedsounds" value="sounds"/>
  </section>
  <section name="graphics">
        <var name="ModelDetail" value=".7"/>
        <var name="x_resolution" value="640"/>
        <var name="y_resolution" value="480"/>
        <var name="mipmapdetail" value="1"/>
        <var name="reflection" value="true"/>

         <section name="hud" >
                   <var name="drawAllTargetBoxes" value="false" />
                   <var name="drawAlwaysITTS" value="false" />
          </section>
  </section>
</variables>

In this examples are two sections , "data" and "graphics".
The graphics sections has a subsection named "hud".

You can set variables to strings, to float or int numbers or to boolean values.

Have a look at your vegastrike.config file, there are a lot more sections/variables that you may want to edit.
